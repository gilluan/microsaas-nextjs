# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This is a MicroSaaS project built with **AWS Amplify Gen 2** as the backend foundation, **Next.js** with App Router for the frontend, and **TypeScript** throughout. The project follows a constitution-driven development approach with strict principles around technology choices and development patterns.

## Core Technology Stack

- **Backend**: AWS Amplify Gen 2 with TypeScript-first configuration in `amplify/backend.ts`
- **Frontend**: Next.js 14+ with App Router, TypeScript strict mode
- **UI Components**: ShadcnUI as the component foundation - customize existing components rather than creating from scratch
- **State Management**: Zustand for global state with Zod schemas for validation
- **Data Layer**: GraphQL generated by Amplify Gen 2 with real-time subscriptions
- **Authentication**: AWS Cognito through Amplify Gen 2 (email/password + Google OAuth)
- **Storage**: Amazon S3 via Amplify Gen 2 with CDN integration
- **Functions**: AWS Lambda via Amplify Gen 2 for custom business logic
- **Payments**: Stripe embedded checkout forms with webhook validation
- **Validation**: Zod schemas at all data boundaries (forms, API responses, store updates)

## Architecture Principles

### 1. AWS Amplify Gen 2 First
- All backend features MUST use Amplify Gen 2 capabilities when available
- Configure everything through `amplify/backend.ts` as single source of truth
- Use TypeScript-first approach for all backend configuration
- Custom backends only when Amplify features are insufficient

### 2. Type Safety Throughout
- TypeScript strict mode everywhere - no `any` types
- Zod schemas for validation at all boundaries
- Combine Zod with Zustand for type-safe state management
- Generate TypeScript types from GraphQL schemas

### 3. Component Consistency
- ShadcnUI components as foundation - customize, don't rebuild
- Use `cn()` utility for conditional styling
- Follow established patterns in `components/ui/`
- Maintain visual consistency across the application

### 4. Security & Best Practices
- Validate all inputs with Zod schemas
- Use Amplify's built-in security features
- Never expose sensitive configuration or API keys
- Implement proper authorization rules (owner, groups, private, public)
- Follow OWASP security guidelines

## Data Layer Patterns

### GraphQL Schema Design
- Use belongsTo/hasMany patterns with proper foreign keys
- Implement authorization rules for multi-tenant SaaS patterns
- Follow GraphQL best practices: proper field naming, nullable vs non-nullable
- Design for pagination, filtering, and sorting consistency

### State Management
- Create domain-specific Zustand stores (auth, user, app state)
- Combine stores with Zod schemas for runtime validation
- Use selectors to prevent unnecessary re-renders
- Follow the pattern: `useStore((state) => state.selector)`

### Real-time Updates
- Use GraphQL subscriptions for live data updates
- Implement proper subscription management (subscribe/unsubscribe)
- Handle connection states and error recovery

## Authentication Patterns

### Cognito Integration
- Configure external providers in `amplify/auth/resource.ts`
- Use proper naming conventions for external providers
- Implement MFA and account recovery workflows
- Handle proper token refresh and session management

### Protected Routes
- Use Next.js middleware for route protection
- Implement proper session management
- Handle loading states during auth checks

## Storage & File Management

- Use pre-signed URLs for secure direct uploads
